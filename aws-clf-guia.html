Modelo de Responsabilidad Compartida

EL más importante del examen
▼
AWS es responsable de la seguridad DE la nube (hardware, software, redes, instalaciones). El cliente es responsable de la seguridad EN la nube (datos, OS, configuraciones, accesos IAM).
📌 Regla mnemotécnicaAWS protege el edificio. Tú pones candado a tu habitación.
💰

Tipos de instancias EC2

Pregunta frecuente
▼
On-Demand: Pagas por hora/segundo, sin compromiso. Ideal para cargas impredecibles.

Reserved: Compromiso 1-3 años. Hasta 75% de descuento. Ideal para cargas estables.

Spot: Usa capacidad no utilizada. Hasta 90% de descuento. Puede interrumpirse. Ideal para workloads tolerantes a fallos.

Savings Plans: Flexibilidad de On-Demand con descuentos de Reserved.
🌍

Infraestructura Global AWS

Regions, AZs, Edge Locations
▼
Región: Área geográfica con 2+ AZs. Elegida por latencia, compliance y costo.

Zona de Disponibilidad (AZ): 1+ data center independiente dentro de una región. Conectados con baja latencia.

Edge Locations: Puntos de presencia para CloudFront y Route 53. Más de 400 en el mundo.
📌 AnalogíaRegión = país, AZ = ciudad, Edge = tienda local de entrega rápida.

API (Application Programming Interface)

Integración de servicios
▼
Una API es un conjunto de reglas y protocolos que permiten que diferentes aplicaciones se comuniquen entre sí. En AWS, la mayoría de las acciones se realizan a través de APIs (vía SDK, CLI o Consola).
📌 EjemploCuando usas boto3 (Python) para crear un bucket S3, estás llamando a la API de S3 detrás de escena.

Beneficios de migrar a AWS

Por qué moverse a la nube
▼
Los 6 beneficios clave:
1. Cambio CapEx → OpEx (pagas por uso)
2. Economías de escala masivas
3. Capacidad flexible sin sobre-provisionar
4. Agilidad para innovar
5. Alcance global en minutos
6. Eliminar data centers propios
📌 Tip examenLos 6 beneficios aparecen casi siempre en preguntas sobre "por qué usar AWS".

Tipos de instancias EC2

On-Demand, Reserved, Spot, Savings Plans
▼
On-Demand: Sin compromiso, máxima flexibilidad.
Reserved (1-3 años): Hasta 75% descuento. Standard o Convertible.
Spot: Capacidad sobrante, hasta 90% descuento, se puede interrumpir.
Dedicated Hosts: Servidor físico dedicado (compliance).
Savings Plans: Compromiso de gasto ($/hora), aplica a EC2, Lambda y Fargate.
📌 ReglaPredecible → Reserved. Impredecible → On-Demand. Tolerante a fallas → Spot. Regulación → Dedicated.

CloudFormation, Terraform

Infraestructura Global de AWS

Regiones, AZs y Edge Locations
▼
Regiones (~33): Áreas geográficas independientes con 3+ AZs. Elegir según: latencia, compliance, costo, disponibilidad de servicios.

AZs (~105): Uno o más data centers con energía, red y conectividad redundante.

Edge Locations (400+): Puntos para CloudFront (CDN) y Route 53.
📌 Multi-AZ vs Multi-RegionMulti-AZ = alta disponibilidad. Multi-Region = recuperación ante desastres + baja latencia global.

AWS vs Cliente
▼
AWS (DE la nube): Hardware, software global, AZs, regiones, edge locations, hypervisor.

Cliente (EN la nube): Datos, cifrado, OS (IaaS), red virtual, firewall, IAM, configuraciones de apps.

Amazon AthenaQuery SQL serverless sobre S3
▼
Servicio de consultas interactivo que analiza datos en S3 usando SQL estándar. Serverless (no hay infraestructura que gestionar), pagas solo por los datos escaneados.
💡 IntegraciónS3 (fuente datos) → Athena (SQL) → QuickSight (visualización)
⚡
Amazon EMRElastic MapReduce – Big Data
▼
Plataforma cloud de Big Data para procesar grandes volúmenes de datos usando Apache Hadoop, Spark, HBase, etc. Gestiona el aprovisionamiento y la configuración de clústeres.
💡 Caso de usoETL masivo, análisis de logs, machine learning con SparkML.
🔧
AWS GlueETL serverless y Data Catalog
▼
Servicio ETL (Extract, Transform, Load) completamente gestionado. Descubre, cataloga y transforma datos de múltiples fuentes. Incluye Data Catalog para metadatos.
💡 IntegraciónRDS/S3 → Glue (ETL) → Redshift (DW) → QuickSight (BI)
🌊
Amazon KinesisStreaming de datos en tiempo real
▼
Suite para capturar, procesar y analizar datos en streaming en tiempo real. Incluye Kinesis Data Streams, Firehose (carga a S3/Redshift), y Data Analytics (SQL sobre streams).
💡 Caso de usoAnálisis de clickstream, monitoreo IoT, logs en tiempo real.
🔎
Amazon OpenSearch ServiceBúsqueda y análisis de logs
▼
Servicio gestionado basado en OpenSearch (fork de Elasticsearch). Ideal para búsqueda full-text, análisis de logs y monitoreo con dashboards Kibana/OpenSearch Dashboards.
💡 IntegraciónCloudWatch Logs → OpenSearch → Dashboard Kibana
📈
Amazon QuickSightBusiness Intelligence y dashboards
▼
Servicio de BI serverless para crear visualizaciones, dashboards interactivos y reportes. Se conecta a Redshift, S3, RDS, Athena y más. SPICE = in-memory engine para rendimiento.
💡 Tip examenQuickSight = visualización BI. No confundir con CloudWatch (monitoring operativo).
🏭
Amazon RedshiftData Warehouse en la nube
▼
Data warehouse en columnas a escala de petabytes. Para análisis OLAP (no OLTP). Redshift Serverless permite análisis sin gestionar clústeres. Muy integrado con S3 (Redshift Spectrum).
💡 OLAP vs OLTPRedshift = análisis histórico. RDS/Aurora = transacciones operacionales.
🔗 Integración de Aplicaciones
🚌
Amazon EventBridgeBus de eventos serverless
▼
Bus de eventos serverless para conectar aplicaciones con datos de tus apps, servicios SaaS y AWS. Define reglas para enrutar eventos a targets (Lambda, SQS, SNS, etc.).
💡 IntegraciónApp genera evento → EventBridge (regla) → Lambda (procesa) → DynamoDB (guarda)
📢
Amazon SNSSimple Notification Service – Pub/Sub
▼
Servicio de mensajería Pub/Sub totalmente gestionado. Envía notificaciones a múltiples suscriptores (email, SMS, Lambda, SQS, HTTP). Un mensaje → muchos destinos (fan-out).
💡 SNS vs SQSSNS = push a múltiples consumidores. SQS = cola donde el consumidor hace pull.
📬
Amazon SQSSimple Queue Service – Cola de mensajes
▼
Cola de mensajes totalmente gestionada para desacoplar componentes. Standard (at-least-once) o FIFO (exactamente-una-vez, orden garantizado). Retención hasta 14 días.
💡 PatrónSNS (fan-out) → SQS queues → Lambda/EC2 (procesadores independientes)
🔄
AWS Step FunctionsOrquestación de workflows
▼
Orquesta funciones Lambda y servicios AWS en workflows visuales (máquinas de estado). Maneja errores, reintentos y paralelismo. Ideal para procesos de negocio complejos.
💡 Caso de usoCheckout e-commerce: verificar stock → cobrar → enviar email → actualizar inventario.
💻 Computación
📦
AWS BatchProcesamiento por lotes gestionado
▼
Ejecuta jobs de batch a cualquier escala. Aprovisiona automáticamente los recursos de cómputo óptimos (EC2 o Spot). Ideal para simulaciones, análisis genómicos, renderizado.
💡 vs LambdaBatch = jobs largos (horas). Lambda = funciones cortas (<15 min).
🖥️
Amazon EC2Elastic Compute Cloud – VMs en la nube
▼
Servidores virtuales en la nube. Elige tipo (t3, m5, c5, r5...), OS, almacenamiento (EBS). IaaS = tú gestionas el OS, parches y middleware. Múltiples modelos de precios.
💡 Familiast = propósito general. c = cómputo optimizado. r = memoria. i = storage. p/g = GPU (ML).
🌱
AWS Elastic BeanstalkPaaS – Despliegue automático
▼
Sube tu código y Beanstalk gestiona automáticamente: EC2, load balancer, auto scaling, actualizaciones. PaaS para Node, Python, Java, .NET, PHP, Ruby, Go, Docker.
💡 Tip examenBeanstalk = PaaS (tú controlas la app, AWS gestiona la infra). Sin costo adicional, solo pagas la infra.
💡
Amazon LightsailVPS simple para beginners
▼
Servidores virtuales simples con precio fijo mensual. Incluye cómputo, almacenamiento, transferencia de datos y DNS. Ideal para WordPress, apps pequeñas, startups.
💡 Tip examenLightsail = simplicidad y precio fijo. EC2 = flexibilidad y control total.
🏢
AWS OutpostsAWS en tu data center
▼
Racks de hardware AWS instalados en tu data center on-premise. Extiende la nube AWS a instalaciones físicas. Mismos servicios y APIs de AWS, pero local para baja latencia o requisitos de soberanía de datos.
💡 Cuándo usarCuando necesitas procesamiento local pero con APIs y herramientas AWS.

Amazon ECRElastic Container Registry
▼
Registro privado de imágenes Docker totalmente gestionado. Almacena, gestiona y despliega imágenes de contenedores. Integrado con ECS y EKS.
💡 Flujo CI/CDCodeBuild (build) → ECR (push imagen) → ECS/EKS (deploy)

Amazon ECSElastic Container Service
▼
Orquestador de contenedores Docker totalmente gestionado. Ejecuta contenedores en EC2 (tú gestionas el servidor) o con Fargate (serverless). Integrado con IAM, ALB, CloudWatch.
💡 ECS vs EKSECS = más simple, AWS-nativo. EKS = Kubernetes estándar, más portabilidad.

🗄️ Base de Datos
⚡
Amazon AuroraBD relacional de alto rendimiento AWS
▼
BD relacional propietaria de AWS, compatible con MySQL y PostgreSQL. 5x más rápida que MySQL estándar, 3x que PostgreSQL. Auto-scaling de storage, replicación multi-AZ automática.
💡 Aurora ServerlessEscala automáticamente según la demanda, incluso a 0. Ideal para cargas intermitentes.
📄
Amazon DocumentDBBD de documentos compatible con MongoDB
▼
Base de datos de documentos JSON totalmente gestionada, compatible con la API de MongoDB. Escala hasta millones de requests/segundo. Ideal para CMS, catálogos, perfiles de usuario.
💡 Tip examenMongoDB en AWS → DocumentDB. Clave-valor → DynamoDB. Grafos → Neptune.
⚡
Amazon DynamoDBNoSQL serverless de baja latencia
▼
BD clave-valor y documentos totalmente serverless. Latencia de single-digit milliseconds a cualquier escala. Auto-scaling, multi-region con Global Tables, backup continuo (PITR).
💡 DAXDynamoDB Accelerator = caché en memoria para DynamoDB. Microsegundos de latencia.
🚀
Amazon ElastiCacheCaché en memoria (Redis/Memcached)
▼
Servicio de caché en memoria totalmente gestionado. Compatible con Redis (más features: pub/sub, persistencia, Lua) y Memcached (más simple). Reduce latencia de BD.
💡 PatrónApp → ElastiCache (si hay hit, responde en microsegundos) → si miss → RDS/DynamoDB
🕸️
Amazon NeptuneBase de datos de grafos
▼
BD de grafos totalmente gestionada. Soporta Gremlin (Apache TinkerPop) y SPARQL. Ideal para redes sociales, recomendaciones, detección de fraude, grafos de conocimiento.
💡 Cuándo usarCuando las relaciones entre entidades son tan importantes como los datos mismos.
🗄️
Amazon RDSRelational Database Service
▼
BD relacionales gestionadas: MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora. AWS gestiona: parches, backups, Multi-AZ. Tú eliges el motor y el tamaño de instancia.
💡 RDS Multi-AZStandby en otra AZ para DR automático. NO mejora rendimiento de lectura (usa Read Replicas para eso).
🔒 Seguridad, Identidad y Cumplimiento
📋
AWS ArtifactReportes de auditoría y compliance
▼
Portal de autoservicio para descargar reportes de seguridad y compliance de AWS (SOC, PCI, ISO). También para revisar y aceptar acuerdos (BAA, NDA) con AWS. Totalmente gratis.
💡 Tip examenCliente necesita reportes de auditoría de AWS → AWS Artifact (no confundir con AWS Audit Manager).
👤
Amazon CognitoAutenticación para apps web y móvil
▼
Gestión de identidad para apps. User Pools (directorio de usuarios, login, MFA) + Identity Pools (federar identidades para acceder a AWS). Soporta OAuth2, OIDC, SAML, social login.
💡 Tip examenAutenticación de usuarios finales de tu app → Cognito. Empleados acceden a AWS → IAM Identity Center.
🔫
Amazon GuardDutyDetección de amenazas con ML
▼
Servicio de detección de amenazas que monitorea continuamente tu cuenta AWS usando ML e inteligencia de amenazas. Analiza CloudTrail, VPC Flow Logs y DNS logs. Sin agentes, sin infra.
💡 IntegraciónGuardDuty detecta → EventBridge → Lambda (remediación automática) → SNS (alerta)
🔑
AWS IAMIdentity and Access Management
▼
Controla quién puede hacer qué en AWS. Usuarios, Grupos, Roles y Políticas (JSON). Principio de mínimo privilegio. IAM es global (no por región). MFA recomendado para root y usuarios.
💡 Componentes Usuario = persona. Grupo = conjunto de usuarios. Rol = identidad temporal (para servicios o federación). Política = permisos JSON.
🔐
AWS KMSKey Management Service
▼
Crea y gestiona claves de cifrado para proteger datos en S3, EBS, RDS, etc. CMKs (Customer Managed Keys) o AWS Managed Keys. FIPS 140-2 Level 2. Auditable vía CloudTrail.
💡 KMS vs CloudHSMKMS = multi-tenant, gestionado por AWS. CloudHSM = hardware dedicado, tú controlas las claves.
🛡️
AWS ShieldProtección DDoS
▼
Shield Standard: Gratuito, automático para todos los usuarios AWS. Protección capa 3 y 4.
Shield Advanced: $3,000/mes. Protección capa 7, DRT (DDoS Response Team), protección de costo.
💡 CombinaciónShield Advanced + WAF + CloudFront = protección completa DDoS.
🧱
AWS WAFWeb Application Firewall
▼
Protege apps web de ataques comunes: SQL injection, XSS, bots. Define reglas para bloquear/permitir tráfico HTTP. Se despliega en CloudFront, ALB, API Gateway, AppSync.
💡 Managed RulesAWS ofrece grupos de reglas gestionadas (OWASP Top 10, Bot Control) listas para usar.
🔏
AWS Secrets ManagerGestión segura de secretos
▼
Almacena y rota automáticamente secretos: contraseñas de BD, API keys, tokens. Rotación automática integrada con RDS. Elimina credenciales hardcodeadas en el código.
💡 vs Parameter StoreSecrets Manager = rotación automática, más caro. SSM Parameter Store = más simple, más barato.
💾 Almacenamiento
🪣
Amazon S3Simple Storage Service – Object Storage
▼
Almacenamiento de objetos ilimitado. Clases: S3 Standard, S3-IA, S3 One Zone-IA, S3 Glacier Instant, Glacier Flexible, Glacier Deep Archive. Versioning, lifecycle policies, replication.
💡 Clases de almacenamientoStandard → IA → One Zone-IA → Glacier Instant → Glacier Flexible → Deep Archive (más barato, más lento).
❄️
Amazon S3 GlacierArchivo de largo plazo
▼
Almacenamiento de largo plazo ultra-económico para datos raramente accedidos. Glacier Instant (ms), Flexible (minutos-horas), Deep Archive (12-48h). Hasta 95% más barato que S3 Standard.
💡 Caso de usoBackups de compliance, archivos médicos, logs de auditoría que debes conservar años.
💽
Amazon EBSElastic Block Store – Disco para EC2
▼
Almacenamiento de bloques persistente para EC2. Como un disco duro en la nube. Tipos: gp3/gp2 (general), io1/io2 (IOPS alto), sc1/st1 (throughput). Replicado dentro de la AZ.
💡 S3 vs EBSEBS = disco para EC2 (bloque, una AZ). S3 = almacenamiento de objetos (global, acceso HTTP).
📁
Amazon EFSElastic File System – NFS gestionado
▼
Sistema de archivos NFS elástico y gestionado para Linux. Se monta en múltiples instancias EC2 simultáneamente (a diferencia de EBS). Escala automáticamente, multi-AZ.
💡 EBS vs EFSEBS = una instancia EC2. EFS = compartido entre muchas instancias simultáneamente.
🔀
AWS Storage GatewayAlmacenamiento híbrido on-premise ↔ AWS
▼
Conecta on-premise con almacenamiento en la nube AWS. Tipos: File Gateway (NFS/SMB → S3), Volume Gateway (iSCSI → EBS/S3), Tape Gateway (VTL → Glacier). Para migraciones híbridas.
💡 Caso de usoEmpresa con sistemas legacy que quiere backup automático a S3 sin cambiar sus aplicaciones.
💾
AWS BackupBackup centralizado y automatizado
▼
Servicio centralizado para automatizar y auditar backups de EC2, EBS, RDS, DynamoDB, EFS, FSx, Storage Gateway. Define políticas de backup unificadas para toda la organización.
💡 Tip examenNecesita backup centralizado multi-servicio → AWS Backup. No necesitas configurar backups individualmente.
🌐 Redes y Entrega de Contenido
🏘️
Amazon VPCVirtual Private Cloud
▼
Red virtual aislada lógicamente dentro de AWS. Define tu espacio de IPs, subnets (públicas/privadas), tablas de rutas, internet gateways, NAT gateways, security groups y NACLs.
💡 Security Groups vs NACLSecurity Groups = stateful, nivel instancia. NACLs = stateless, nivel subnet.
⚡
Amazon CloudFrontCDN global – Entrega de contenido
▼
Red de entrega de contenido (CDN) con 400+ edge locations. Cachea contenido estático y dinámico cerca del usuario. Integrado con S3, ALB, EC2, Lambda@Edge. Reduce latencia global.
💡 IntegraciónS3 (origen) → CloudFront (distribución) → Usuario final (edge location más cercana)
🗺️
Amazon Route 53DNS gestionado y registro de dominios
▼
DNS escalable y altamente disponible + registro de dominios. Políticas de enrutamiento: Simple, Weighted, Latency, Failover, Geolocation, Geoproximity, Multivalue. Health checks integrados.
💡 FailoverRoute 53 detecta que el primary está caído (health check) y redirige tráfico al secondary automáticamente.
🔌
AWS Direct ConnectConexión dedicada on-premise ↔ AWS
▼
Conexión de red privada dedicada (fibra) entre tu data center y AWS. Evita internet público → menor latencia, mayor ancho de banda, más consistente. Velocidades de 1Gbps a 100Gbps.
💡 Direct Connect vs VPNDX = dedicado, caro, sin internet. VPN = sobre internet, cifrado, rápido de configurar.
🚪
Amazon API GatewayCrear y gestionar APIs
▼
Crea, publica y gestiona APIs REST, HTTP y WebSocket a cualquier escala. Integrado con Lambda (arquitectura serverless), EC2, servicios AWS. Throttling, caching, autorización.
💡 Patrón serverlessCliente → API Gateway → Lambda → DynamoDB (backend completamente serverless)
🤖 Machine Learning
🧠
Amazon SageMaker AIPlataforma ML end-to-end
▼
Plataforma completa para construir, entrenar y desplegar modelos ML. Jupyter notebooks gestionados, algoritmos preconstruidos, entrenamiento distribuido, endpoints de inferencia. MLOps integrado.

Amazon CloudWatchMonitoreo y observabilidad

Servicio central de monitoreo: métricas (CPU, red, custom), logs (CloudWatch Logs), dashboards, alarmas, eventos. Retención de logs configurable. Container Insights, Lambda Insights.

AWS CloudTrailAuditoría de llamadas API
▼
Registra TODAS las llamadas API realizadas en tu cuenta AWS (quién, qué, cuándo, desde dónde). Habilitado por defecto, retención de 90 días. Para compliance, auditoría forense y seguridad.
💡 CloudTrail vs CloudWatchCloudTrail = quién hizo QUÉ (auditoría). CloudWatch = cómo está FUNCIONANDO (monitoreo).

Crea y gestiona recursos AWS con templates YAML/JSON. Define stacks de infraestructura reproducibles. Soporte para drift detection, rollback automático, nested stacks.

Analiza tu cuenta AWS y da recomendaciones en 5 pilares: Optimización de costos, Rendimiento, Seguridad, Tolerancia a fallos, Límites de servicio. Basic: 7 checks. Business/Enterprise: todos.

AWS OrganizationsGestión de múltiples cuentas
▼
Agrupa y gestiona múltiples cuentas AWS. Políticas de control de servicio (SCP) para establecer límites en toda la organización. Facturación consolidada, descuentos por volumen.

AWS Auto ScalingEscalado automático de recursos
▼
Ajusta automáticamente la capacidad de recursos según la demanda. Escalado dinámico (basado en métricas CloudWatch) y predictivo (ML). Aplica a EC2, ECS, DynamoDB, Aurora.

AWS Snow FamilyTransferencia física de datos masivos
▼
Snowcone: 8TB, portátil. Snowball Edge: 80TB, compute y storage. Snowmobile: 100PB, camión. Cuando la red es demasiado lenta o costosa para transferir datos.

AWS DMSDatabase Migration Service
▼
Migra bases de datos hacia AWS con mínimo downtime. Soporta migraciones homogéneas (MySQL→RDS MySQL) y heterogéneas (Oracle→Aurora). CDC (Change Data Capture) para replicación continua.
💡 + SCTDMS + Schema Conversion Tool (SCT) = migración completa de BD heterogéneas (Oracle → PostgreSQL).

▼

💡 FlujoDiscovery Service (inventario) → Migration Hub (plan) → Application Migration Service (migrar)

AWS LambdaFunciones serverless basadas en eventos
▼
Ejecuta código sin gestionar servidores. Triggers: API Gateway, S3, DynamoDB Streams, EventBridge, SQS, SNS y más. Límite 15 min. Escala automáticamente. Pagas por ms de ejecución.
💡 ConcurrenciaLambda puede ejecutar miles de instancias simultáneas. Reserva concurrencia para funciones críticas.
🚀
AWS FargateContenedores serverless
▼
Motor de cómputo serverless para contenedores con ECS y EKS. No gestionas EC2 ni clústeres. Defines CPU/memoria por tarea y pagas por lo usado. Ideal para cargas intermitentes.
💡 Fargate vs LambdaFargate = contenedores largos (sin límite de tiempo). Lambda = funciones cortas (max 15min).

AWS CodeBuildCI – Build y test automatizados
▼
Servicio de integración continua (CI) totalmente gestionado. Compila código, ejecuta pruebas y produce artefactos listos para desplegar. Sin servidores de Jenkins que gestionar.

AWS CodePipelineCD – Pipeline de entrega continua
▼
Orquesta pipelines de CI/CD completos. Define etapas: source → build → test → deploy. Integra GitHub, Bitbucket, S3, CodeBuild, CloudFormation, ECS, Lambda y más.
💡 Tip examenCodePipeline = orquestador del flujo. CodeBuild = quien construye. CodeDeploy = quien despliega.

AWS X-RayTrazabilidad distribuida
▼
Analiza y depura aplicaciones distribuidas. Traza requests a través de microservicios, Lambda, EC2, ECS. Service map visual, análisis de latencia, identificación de cuellos de botella.
